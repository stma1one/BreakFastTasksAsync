# BreakFastTasksAsync - מדריך פעולות אסינכרוניות (ל תלמידי תיכון)

ברוכים הבאים לפרויקט BreakFastTasksAsync. המטרה של ה-README הזה היא להסביר בצורה ברורה ופשוטה לתלמידי תיכון כיצד לעבוד עם פעולות אסינכרוניות ב-C# (Tasks, async/await), להראות דוגמאות קוד מעשיות, ולהציע תרגילים עם פתרונות. כל הדוגמאות ב-README מניחות שפה: C#.

---

מה תלמד כאן
- מה זו אסינכרוניות ולמה היא חשובה
- מהו Task וכיצד עובדים עם async/await
- איך להריץ מספר משימות במקביל (WhenAll, WhenAny)
- טיפול בביטול (Cancellation) וטיפול בשגיאות (Exceptions)
- דוגמאות פשוטות להרצה בפרויקט
- תרגילים עם פתרונות להצמדה

דרישות מקדימות
- ידע בסיסי ב-C# (פונקציות, מחלקות, עבודה עם console)
- .NET SDK מותקן (גרסה מודרנית - .NET 6, 7 או 8 מומלץ)

הסבר קצר: סינכרוני מול אסינכרוני
- סינכרוני: פונקציה מבצעת את כל עבודתה לפני שהיא מחזירה שליטה למקרא (caller).
- אסינכרוני: פונקציה יכולה להתחיל עבודה שיכולה להתרחש ברקע ולהחזיר מיד שליטה, כדי לא לחסום את חוט הביצוע הראשי (כמו ממשק משתמש או שרת).

ב-C#, המודל הנפוץ הוא מבוסס על Task ו-async/await.

Task ו-async/await - העיקר
- Task מייצג עבודה שיכולה להסתיים בעתיד. Task<TResult> מחזיק תוצאה.
- מילת המפתח async מצביעה שהפונקציה יכולה להכיל await.
- await מחכה באופן אסינכרוני ל-Task בלי לחסום את ה-thread.

דוגמה בסיסית
```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        Console.WriteLine("התחל");
        await DoWorkAsync();
        Console.WriteLine("סיים");
    }

    static async Task DoWorkAsync()
    {
        Console.WriteLine("עבודה אסינכרונית מתחילה...");
        await Task.Delay(1000); // מחכה דמה של שנייה בלי לחסום את ה-thread
        Console.WriteLine("עבודה אסינכרונית הסתיימה.");
    }
}
```
הסבר: Task.Delay מדמה פעולה של פעולה שיכולה לקחת זמן (כגון קריאת רשת). השורה await Task.Delay(1000) תחכה באופן אסינכרוני לשניה.
